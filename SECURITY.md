# 보안 정책 (Security Policy)

> ⚠️ **비공식(unofficial) 프로젝트** — 토스증권과 무관하며 상표/엔도르스먼트와도 무관합니다. 이 소프트웨어는 **있는 그대로** 제공되며, 사용에 따른 책임은 사용자에게 있습니다.

## 지원 버전

`main` 브랜치와 최신 `0.0.x` 릴리스에 대해서만 보안 수정을 제공합니다.

## 취약점 제보

**공개 이슈로 올리지 마세요.** GitHub 의 **Security → "Report a vulnerability"**(비공개 보안 권고)로 제보해 주세요. 가능한 한:

- 재현 절차 / PoC
- 영향 범위(어느 패키지·모드·트랜스포트)
- 영향받는 버전

수정·공개 일정은 제보 내용에 따라 비공개로 협의합니다.

## 자격증명(키) 안전 — 이 프로젝트의 핵심

이 레포의 존재 이유 자체가 "증권계좌 키를 안전하게 다루기"입니다. 운영 시 다음을 지키세요.

- **`.env`·키를 절대 커밋하지 마세요.** `deploy/.env` 는 `.gitignore`·`.dockerignore` 로 막혀 있습니다. 실수로 커밋했다면 **즉시 키를 폐기·재발급**하세요(히스토리에서 지워도 노출된 키는 죽은 키로 취급).
- **자격증명은 서버 env 로만 보관.** 토스 `CLIENT_ID`/`CLIENT_SECRET` 은 브라우저·MCP 클라이언트·로그·Redis 어디에도 노출되지 않습니다. 감사 로그(JSONL)에도 기록하지 않습니다.
- **기본 모드는 `paper`(실주문 0).** 실거래는 `TOSSINVEST_MODE=live` **+** `TOSSINVEST_ALLOW_LIVE=1` 이중 게이트가 둘 다 있어야 켜집니다.
- **HTTP 원격 모드는 단일 테넌트 + bearer 필수.** `TOSSINVEST_AUTH_TOKEN` 없이는 부팅을 거부합니다. 토큰은 **MCP 엔드포인트 인증일 뿐** 토스 자격증명이 아니며, 충분히 긴 랜덤 값을 쓰고 남과 공유하지 마세요. 토스 약관이 자격증명의 제3자 제공·공유를 제한합니다.
- **`live` + 자동승인 클라이언트 금지.** 2단계 주문(preview→place)은 사람이 각 호출을 승인하는 전제입니다. 자동승인을 켜면 LLM 이 한 턴에 연달아 체결할 수 있습니다. `LIVE_CONFIRM_MIN_DELAY_SEC` 로 최소 간격을 강제하세요.

## 무결성 축 — 프롬프트 인젝션과 이 서버의 경계

위 조항은 전부 **기밀성·인가**(키를 지키고, 실주문을 이중 게이트로 막는다)를 다룹니다. 별개의 축이 있습니다: **무결성** — 툴 설명(description)이나 툴 반환값에 심긴 지시문이 클라이언트 에이전트의 컨텍스트를 오염시키는 공격(tool description / output injection)입니다. 이 축은 구조적으로 **서버가 대신 막아줄 수 없습니다.** 인젝션의 표적은 서버가 아니라 서버의 출력을 읽는 **에이전트의 판단**이기 때문입니다.

- **방어 지점은 클라이언트입니다.** 인젝션 인식이 있는 최신 모델을 쓰고, 자동승인을 끄고(위 `live` 조항), 클라이언트 측 툴 게이팅을 켜세요. 서버 방어(bearer·2단 게이트·guardrail)는 이 축과 직교하므로 인젝션을 걸러내지 못합니다.
- **서버 측 defense-in-depth.** 이 서버는 툴 설명·반환값을 **데이터만** 담고 지시문·긴급성 어휘를 넣지 않도록 유지합니다. 새 툴을 추가할 때 설명에 "먼저 …를 호출하라" 류의 지시를 넣지 마세요 — 정상 설명과 인젝션을 구분하는 유일한 방어선이 흐려집니다.
- **스캐너 한계 인지.** description 인젝션 전용 스캐너도 **분할(여러 툴에 조각내 심는) 인젝션과 output 인젝션은 놓칩니다.** 스캔 통과 = 안전 아님.

자세한 안전모델은 [`pytossinvest-mcp/README.md`](pytossinvest-mcp/README.md) 참고.
